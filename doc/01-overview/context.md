# Contexte

L’application monolithique actuelle présente plusieurs limites :

- lenteur aux heures de pointe  
- difficulté à intégrer de nouveaux canaux (mobile, web, SMS)  
- faible évolutivité  
- impact global lors d’une panne

Pour répondre à ces enjeux, la DSI valide la transition vers une architecture **microservices**.

Sources du cahier des charges : fourni en TP Architecture Microservices.
